Dashboard Index

Each of these prototype dashboard interfaces is an example of combining a different set of third-party solutions (genome trackers, 3d renders, etc.) with the genomedashboard controller code as implemented in the genome dashboard javascript and python libraries.

Dashboard Objectives

  • Validate compatibility between all possible candidate solutions and the genome dashboard libraries.
  • Identify performance differences between candidate solutions.
  • Normalize the interactions between these vendor solutions.
